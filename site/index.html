The core trick

A button press becomes a keystroke.

There is no fragile automation layer. A UI action writes raw bytes to a real pseudo-terminal (PTY). The shell and the agent read those bytes as if they came from your keyboard.

1 · You click

UI action fires

A button calls invoke("pty_write", { id, data }) in the web UI.

2 · Rust writes

Bytes hit the PTY

The Rust backend writes those bytes to the PTY's stdin — real input.

3 · Agent reads

Renders back in xterm

Output streams as pty-output events and paints in the terminal.

Same bridge powers everything: real keystrokes, button injection, agent launches, and one-line questions from the canvas all travel the one path.

Attention alerts

OctiqFlow scans terminal output for notify escape codes. When an agent needs you, its tab badges and a banner appears. Click to jump straight there.

OSC 9 / 777 / Kitty · octiq-notify

Agent-session resume

Restart the app and re-attach a tab to its prior agent session — claude --resume / codex resume. A best-effort hook captures the session id while it runs.

survives restart

The living canvas

A pane beside the terminal renders HTML or Markdown the agent writes — a plan, a diagram, a decision log — and updates it live as you talk.

Spotlight · Canvas

Watch the plan grow next to the chat.

While you talk with the agent in a terminal, it draws a plan, a spec, or a diagram onto a pane beside it — and keeps rewriting the same document as decisions land. No IPC socket; just a file on disk and a watcher.

  • Renders Markdown or raw HTML in a sandboxed iframe — agent code can animate but never reaches your machine.
  • Highlight to ask: select text on the canvas and send a one-line question straight to the live terminal.
  • Lives under your home folder, so it never dirties the repo and survives a restart.

Under the hood

Native where it counts.

A Tauri shell, a Rust backend, and a no-bundler web UI rendering terminals on the GPU.

Shell

Tauri 2

One cross-platform desktop app for macOS, Windows, and Linux — small, native, no Electron.

Backend

Rust + portable-pty

Multi-PTY manager, git reads, file browser, layout store, and the OSC attention scanner.

Terminal

xterm.js · WebGL

Vendored, no CDN. WebGL rendering avoids the ghosted glyphs the DOM renderer leaves after reflow.

UI

Vanilla ES modules

No bundler, no framework. Modules talk through window events; the whole UI runs straight from disk.

Get started

Clone, install, run.

OctiqFlow builds from source with the Tauri toolchain. Rust, Node, and pnpm are the only requirements.

terminal 
# 1 — clone the repo
git clone https://github.com/kyson8101/octiq-flow.git
cd octiq-flow

# 2 — install the Tauri CLI (the only JS dependency)
pnpm install

# 3 — run the dev window with hot reload
pnpm tauri dev

# …or build the release .app / installer
pnpm tauri build
  • R
    Rust

    Powers the PTY bridge and every backend command. Install via rustup or Homebrew.

  • N
    Node + pnpm

    Only used to fetch and run the Tauri CLI — the web UI itself ships with no build step.

  • A
    A CLI agent

    Have claude or codex on your PATH — OctiqFlow launches them in a login shell.
